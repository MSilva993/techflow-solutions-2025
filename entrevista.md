## Entrevista com o Cliente – Levantamento de Requisitos

1. **Qual é o principal objetivo do sistema?**  
   Acompanhar o andamento das tarefas da equipe em tempo real, com foco em produtividade e agilidade.

2. **Quais funcionalidades são essenciais para o sistema?**  
   Criar tarefas, listar tarefas, marcar como concluídas e excluir tarefas. Tudo deve ser simples e direto.

3. **Como a equipe costuma organizar o trabalho atualmente?**  
   Usamos planilhas e mensagens no WhatsApp, mas isso gera confusão e retrabalho.

4. **Vocês precisam de controle de acesso ou login?**  
   Não neste momento. O sistema será usado internamente por uma equipe pequena.

5. **É importante visualizar o status das tarefas?**  
   Sim, queremos saber rapidamente o que está pendente ou concluído.

6. **Vocês gostariam de priorizar tarefas?**  
   Sim, mas pode ser uma funcionalidade futura. Por enquanto, queremos focar no básico.

7. **Como vocês gostariam de acompanhar o progresso?**  
   Um quadro visual simples, com colunas representando o andamento das tarefas.  
   *(No projeto, isso foi implementado usando o Board simples do GitHub Projects.)*

8. **Vocês precisam de relatórios ou gráficos?**  
   Não por enquanto. O foco é na execução e acompanhamento diário.

9. **Qual é o prazo esperado para entrega?**  
   O mais breve possível, mas com qualidade e validação das funcionalidades principais.

10. **Vocês esperam que o sistema evolua no futuro?**  
    Sim. Se funcionar bem, podemos incluir login, filtros avançados, prioridades e integração com outros sistemas.

---

Este levantamento foi utilizado como base para definir o escopo inicial, criar as Issues no 
GitHub e organizar o Board do projeto.
